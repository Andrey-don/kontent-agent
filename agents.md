# agents.md — Команда агентов: Бот-копирайтер

> Проект: Telegram-бот, который пишет в стиле автора на основе загруженных файлов
> Стек: Python + Telegram Bot API + OpenRouter

## Схема работы

```
Пользователь (Telegram)
        ↓
   Диспетчер (классифицирует запрос)
        ↓
   Оркестратор (координирует)
   ↙    ↓     ↓      ↓      ↓
Ресёр- Архи- Копи- Критик Конт.
чер   тектор райт.        планер
              ↓
           Редактор
              ↓
          Тестировщик
              ↓
   Результат → Пользователь
```

---

## Полный цикл генерации поста

1. **Диспетчер** → определяет тип задачи (GENERATE / EDIT / PLAN / ANALYZE)
2. **Ресёрчер** → собирает материал по теме
3. **Архитектор** → планирует структуру поста
4. **Копирайтер** → пишет черновик в голосе автора
5. **Критик** → оценивает и даёт замечания
6. **Редактор** → правит с учётом замечаний
7. **Тестировщик** → финальная проверка по чек-листу

## Чек-лист тестировщика

- [ ] Длина 150–300 слов
- [ ] Первые 1–2 предложения — крючок (результат или провокация)
- [ ] Короткие предложения, разговорный стиль
- [ ] Пост про клиента, не про инструмент
- [ ] Провокационный вопрос перед CTA
- [ ] Кодовое слово в CTA (заглавными)
- [ ] 3–5 эмодзи

---

*Зачем столько агентов? ИИ плохо понимает многосоставные задачи. Когда каждый агент отвечает за одну задачу — результат точнее.*
